# Debug Command

使用 `/debug` 来触发系统化调试流程。

## 适用场景

当您遇到以下情况时，使用此命令：

- **Bug 定位和修复** - 发现代码中的错误
- **测试失败排查** - 测试用例失败，需要找出原因
- **意外行为分析** - 代码行为与预期不符
- **性能问题调查** - 系统运行缓慢或资源占用高
- **构建失败** - 编译或打包过程出错
- **集成问题** - 组件之间通信或协作出现问题

## 使用方式

直接在对话中输入：
```
/debug
```

或者：
```
/debug 我的代码出现了 XXX 错误
```

## 调试流程

此命令将触发 `systematic-debugging` 技能，包含以下阶段：

| 阶段 | 内容 |
|------|------|
| **Phase 1: 根因调查** | 读取错误信息、复现问题、检查最近变更、收集证据 |
| **Phase 2: 模式分析** | 查找工作示例、对比差异、理解依赖 |
| **Phase 3: 假设验证** | 形成假设、最小化测试、验证结论 |
| **Phase 4: 实施修复** | 创建失败测试、实施单一修复、验证结果 |

## 仅分析模式

如果您只想获取分析结果而不自动修复，可以说：

- "告诉我问题在哪里，并让我判断是否需要修复"
- "分析一下，不要改代码"
- "只做根因分析"

## 相关技能

- `verification-before-completion` - 验证修复是否有效
- `test-driven-development` - 创建失败的测试用例
