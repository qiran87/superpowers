# Bad Case 1: Extra Fields Detection

**检测目标:** 发现代码中使用了协议文档中未定义的字段

## 检测原理

```
协议文档定义的白名单字段集合
    ↓
提取代码中实际使用的字段
    ↓
集合差集运算: used_fields - allowed_fields
    ↓
发现越界字段 → 报告违规
```

## 检测规则

### 前端代码检测

**检测目标:**
- TypeScript/JavaScript 字段访问
- React props 和 state
- API 响应字段解构

**检测模式:**

**Pattern 1: 对象属性访问**
```typescript
// ❌ 越界使用
const { phone, address } = user;  // phone, address 可能未定义

// ✅ 正确使用
const { id, name, email } = user;  // 所有字段都在协议中
```

**Pattern 2: 字段路径访问**
```typescript
// ❌ 越界使用
user.profile.avatarUrl  // profile 或 avatarUrl 可能未定义

// ✅ 正确使用
user.name  // name 在协议中定义
```

**Pattern 3: API 响应解构**
```typescript
// ❌ 越界使用
const { data: { user: { fullName } } } = await api.getUser();
// fullName 可能未定义

// ✅ 正确使用
const { data: { user: { id, email } } } = await api.getUser();
```

### 后端代码检测

**检测目标:**
- API 响应字段
- DTO/Data Transfer Object 字段
- 数据库查询结果映射

**检测模式:**

**Pattern 1: 响应对象构建**
```typescript
// ❌ 越界添加字段
return {
  id: user.id,
  name: user.name,
  extraField: user.custom_data  // extraField 不在协议中
};

// ✅ 正确构建
return {
  id: user.id,
  name: user.name
};
```

**Pattern 2: 字段映射**
```typescript
// ❌ 越界映射
const dto = {
  userId: user.id,
  userName: user.name,
  userPhone: user.phone  // phone 不在协议中
};

// ✅ 正确映射
const dto = {
  userId: user.id,
  userName: user.name
};
```

### 数据库代码检测

**检测目标:**
- INSERT 语句的字段列表
- UPDATE 语句的 SET 子句
- ORM 模型的属性

**检测模式:**

**Pattern 1: INSERT 字段**
```sql
-- ❌ 越界插入
INSERT INTO users (id, name, phone) VALUES (?, ?, ?);
-- phone 不在 schema 中

-- ✅ 正确插入
INSERT INTO users (id, name, email) VALUES (?, ?, ?);
```

**Pattern 2: ORM 字段赋值**
```typescript
// ❌ 越界字段
await User.create({
  id: generateId(),
  name: 'Alice',
  customField: 'value'  // customField 不在 schema
});

// ✅ 正确字段
await User.create({
  id: generateId(),
  name: 'Alice'
});
```

## 检测算法

### 步骤 1: 提取协议字段白名单

```python
# 伪代码
def extract_protocol_fields():
    allowed_fields = set()

    # 从领域模型提取
    domain_models = read_file('docs/project-analysis/03-backend-domains.md')
    allowed_fields.update(extract_entity_fields(domain_models))

    # 从 API 定义提取
    api_docs = read_file('docs/project-analysis/02-backend-apis.md')
    allowed_fields.update(extract_api_fields(api_docs))

    # 从数据库架构提取
    db_schema = read_file('docs/project-analysis/04-database-schemas.md')
    allowed_fields.update(extract_table_columns(db_schema))

    return allowed_fields
```

### 步骤 2: 提取代码使用的字段

```python
# 伪代码
def extract_code_fields(code_files):
    used_fields = set()

    for file in code_files:
        if file.endswith(('.ts', '.tsx', '.js', '.jsx')):
            # 解析 AST
            ast = parse(file)

            # 提取对象属性访问
            used_fields.update(extract_property_access(ast))

            # 提取解构赋值
            used_fields.update(extract_destructuring(ast))

        elif file.endswith('.sql'):
            # 解析 SQL
            used_fields.update(extract_sql_columns(file))

    return used_fields
```

### 步骤 3: 检测越界字段

```python
# 伪代码
def detect_extra_fields(allowed_fields, used_fields):
    extra_fields = used_fields - allowed_fields

    violations = []
    for field in extra_fields:
        location = find_field_location(field)
        violations.append({
            'field': field,
            'file': location.file,
            'line': location.line,
            'severity': 'CRITICAL',
            'message': f"Field '{field}' not defined in protocol documentation"
        })

    return violations
```

## 报告格式

### 单个违规报告

```markdown
### ❌ Issue #[N]: Extra field used

- **Location:** `src/components/UserProfile.tsx:45`
- **Code Snippet:**
  ```typescript
  const { phone, address } = user;
  ```
- **Problem:** Fields `phone`, `address` not defined in:
  - `docs/project-analysis/03-backend-domains.md` (User entity)
  - `docs/project-analysis/04-database-schemas.md` (users table)
- **Impact:** Code depends on undefined fields, may cause runtime errors
- **Fix Options:**
  1. Remove field usage if not needed
  2. Add fields to protocol documentation (requires design update)
  3. Check if fields are from parent object (e.g., user.profile.phone)

**Severity:** CRITICAL

**Rationale:** Using undefined fields breaks the contract between implementation and design, leading to:
- Runtime errors when fields are missing
- Inconsistent data across components
- Maintenance burden (field meaning unclear)
```

## 特殊情况处理

### 框架自动生成的字段

**不报告为违规的字段:**
- `id` (auto-generated primary key)
- `createdAt`, `updatedAt` (ORM timestamps)
- `version` (optimistic locking)

**报告格式:**
```markdown
### ℹ️ Info: Framework-generated field

- **Field:** `createdAt`
- **Location:** `src/models/User.ts:12`
- **Note:** Auto-generated by ORM, not in protocol
- **Recommendation:** Document in protocol if used by business logic
```

### 可选字段

**协议中标记为可选的字段:**
- 如果代码检查字段存在性 → OK
- 如果代码假设字段总是存在 → 可能违规

**示例:**
```typescript
// ❌ 假设可选字段总是存在
const fullName = user.profile.fullName;  // profile 可能为 null

// ✅ 正确处理可选字段
const fullName = user.profile?.fullName;
```

### 嵌套对象字段

**检测嵌套路径:**
```typescript
// ❌ 中间对象未定义
user.profile.avatarUrl  // profile 未在协议中

// ✅ 检查协议定义
// 如果协议定义: User { profile: Profile { avatarUrl } }
// 则访问 user.profile.avatarUrl 是合法的
```

## 修复建议

### 修复策略 1: 删除字段使用

**适用场景:** 字段不是必需的

**修复步骤:**
1. 删除越界字段的使用
2. 验证代码仍然正常工作
3. 更新相关测试

**示例:**
```typescript
// Before
const { phone, address } = user;

// After
const {} = user;  // 移除不需要的字段
```

### 修复策略 2: 更新协议文档

**适用场景:** 字段是必需的，但遗漏了

**修复步骤:**
1. 更新技术方案设计文档
2. 更新 `docs/project-analysis/03-backend-domains.md`
3. 更新相关 API 文档
4. 可能需要数据库迁移
5. 更新相关测试

**示例:**
```markdown
# 更新 docs/project-analysis/03-backend-domains.md

## User Entity

**Fields:**
- id: UUID
- name: String
- email: String
- phone: String **[NEW]**
- address: Address **[NEW]**
```

### 修复策略 3: 使用父对象

**适用场景:** 字段属于父对象，但直接访问

**修复步骤:**
1. 检查协议中是否定义了父对象
2. 使用正确的字段路径
3. 更新相关代码

**示例:**
```typescript
// Before
const phone = user.phone;  // ❌ 直接访问

// After
const phone = user.profile.phone;  // ✅ 通过父对象访问
```

## 检测优先级

| 代码类型 | 优先级 | 理由 |
|---------|--------|------|
| 前端组件 | HIGH | 直接影响用户体验 |
| API 接口 | CRITICAL | 影响前后端契约 |
| 数据库操作 | CRITICAL | 可能导致运行时错误 |
| 工具函数 | MEDIUM | 影响有限 |
