## 关键修复

### 问题：grep -c 退出码导致双重输出

**原始错误代码:**
```bash
IN_PROGRESS=$(grep -cE "pattern" file || echo 0)
# 问题：grep 没有匹配时返回 1 退出码和 0 输出
# 然后 || echo 0 再输出 0
# 结果：变量变成 "0\n0" (包含换行符)
```

**修复后代码:**
```bash
IN_PROGRESS=$(grep -E "pattern" file 2>/dev/null | grep -E "pattern" | wc -l | tr -d ' ')
# 解决方案：
# 1. 使用管道和 wc -l 计数（避免 grep -c 的退出码问题）
# 2. tr -d ' ' 去除可能的空白字符
# 3. 整个结果在子shell 中执行，避免退出码问题
```
